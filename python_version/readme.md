# 项目：相机 Rig 标定可视化工具

## 目标
构建一个工具，从 `device_calibration.xml` 文件中重建相机 Rig 的空间布局。这个xml文件是高通的一个标定文件。

## 核心需求

1. **解析 device_calibration.xml**  
   读取并解析标定文件中的相机和 IMU 参数。

2. **以 trackingA 作为世界坐标系原点**  
   将 `trackingA` 相机的位姿设为世界坐标系的基准（即其旋转矩阵为单位矩阵，平移向量为零）。

3. **重建相机位姿**  
   根据标定数据计算每个相机相对于世界坐标系的位姿（旋转矩阵 $R$ 和平移向量 $t$）。

4. **重建 IMU 位姿（SFConfig）**  
   从 `SFConfig` 节点中提取 IMU 相对于参考坐标系（通常是 `trackingA`）的位姿。

5. **渲染 3D 可视化场景**  
   使用 `matplotlib` 的 3D 绘图功能展示整个相机 Rig 的空间结构。

6. **显示相机视锥体（Frustum）**  
   根据相机的内参（焦距、主点、分辨率）和模型（如针孔或鱼眼），绘制每个相机的视锥体。

7. **显示坐标轴**  
   为每个相机和 IMU 绘制局部坐标系的三轴（X-红，Y-绿，Z-蓝）。

9. **标注相机名称**  
   在 3D 图中标注每个相机的名称（如 `trackingA`, `trackingB` 等）。

10. **支持添加虚拟相机**  
    允许用户定义并可视化额外的虚拟相机位姿，用于仿真或测试。该相机可能只有坐标系关系并没有实际的坐标轴关系


## 可视化内容

- 相机光心位置（用点标记）
- 相机局部坐标轴（三色线段）
- 相机视锥体（金字塔形线框）
- IMU 局部坐标轴

## 可扩展性（未来改进方向）

- **交互式相机编辑**：允许用户拖拽调整相机位姿。
- **标定编辑 GUI**：提供图形界面修改标定参数。
- **基线叠加显示**：在图中高亮显示关键相机对之间的基线。
- **视场角（FOV）编辑**：动态调整并更新视锥体形状。
- **导出新标定 XML**：将修改后的位姿保存为新的 `device_calibration.xml` 文件。

---

> 该项目旨在为多相机系统（如 VR/AR 头显、机器人视觉系统）提供直观的标定结果验证与调试能力。
